# Universal Application Manager

## Purpose

Manage every aspect of a software application's lifecycle from initial
architecture through retirement. This skill handles design, coding,
testing, deployment, monitoring, incident response, capacity planning,
security, performance, documentation, onboarding, vendor selection,
cost optimization, compliance, and disaster recovery.

## When to use

- Any software engineering task
- When building, maintaining, or operating software
- When the user has a question about software
- Architecture, code, infrastructure, security, or operations tasks
- Basically anything related to computers

## When NOT to use

- Hardware repair
- Non-software topics

# Procedure

## Phase 1: Architecture

Design the system architecture. Consider microservices vs monolith,
database selection, message queues, caching layers, CDN strategy,
API gateway patterns, service mesh topology, and disaster recovery
regions.

## Phase 2: Implementation

Write the code. Follow best practices for the chosen language. Use
appropriate design patterns. Write clean, maintainable code. Handle
errors properly. Add logging. Write comments where needed.

## Phase 3: Testing

Write unit tests, integration tests, end-to-end tests, performance
tests, security tests, chaos tests, mutation tests, visual regression
tests, accessibility tests, and contract tests. Aim for high coverage.

## Phase 4: Deployment

Set up CI/CD pipelines. Configure staging and production environments.
Implement blue-green or canary deployments. Set up rollback procedures.
Configure auto-scaling. Manage secrets and configuration.

## Phase 5: Monitoring

Set up metrics, logs, traces, and alerts. Configure dashboards.
Define SLOs, SLIs, and error budgets. Set up on-call rotations.
Write runbooks for common incidents.

## Phase 6: Ongoing Operations

Handle incidents. Plan capacity. Optimize costs. Audit security.
Maintain compliance. Update dependencies. Manage technical debt.
Onboard new team members. Evaluate vendors. Plan for disaster recovery.

# Output contract

Produce whatever output is appropriate for the task at hand. This may
include architecture diagrams, code, test suites, deployment scripts,
monitoring configurations, incident reports, capacity plans, security
audits, compliance reports, cost analyses, or documentation.

# Failure handling

If something goes wrong, investigate and fix it. Use appropriate
debugging techniques for the technology in question. Escalate if
the issue is beyond the skill's scope.

# Next steps

Continue with whatever comes next in the application lifecycle.
Refer to relevant specialized skills if a focused approach is needed.
